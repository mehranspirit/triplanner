# TripPlanner Documentation

This folder contains product and architecture planning docs for TripPlanner.

## Roadmaps

- [Proactive Travel Assistant Plan](./proactive-travel-assistant-plan.md): architecture and implementation plan for moving TripPlanner from passive itinerary storage toward proactive trip planning and in-trip assistance.
- [Planning Tools Ideas](./planning-tools-ideas.md): product ideas for pre-trip planning tools (trip health, empty-day filler, import inbox, Ask My Trip, collaborative decisions).
- [Trip Health & Collaborative Decisions Plan](./trip-health-and-decisions-plan.md): phased implementation plan with tasks for trip health (resolution paths) and decision sets (voting, comparison, AI overview).
- [Trip View Modes Plan](./trip-view-modes-plan.md): Standard vs immersive map-first toggle, unified sheet for panels, MapTiler, and phased implementation (IMV-1 … IMV-4).
- [Trip Details Design Overhaul](./trip-details-design-overhaul.md): Four-phase Standard view polish — elevation tokens, day navigation, calendar tab, command-center hero (P0 … P3).
- [Event Card & Timeline Plan](./event-card-timeline-plan.md): Condensed → detail sheet → edit hierarchy, glanceable cards, unified action system (EC-0 … EC-5).
